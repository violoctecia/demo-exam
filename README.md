# Запуск проекта

## Подготовка базы данных

1. Создайте базу данных в PostgreSQL с именем `demo_exam`
2. Создайте файл .env в корне проекта рядом с существующим `.env.example`
3. Скопируйте содержимое `.env.example` в `.env`

## Запуск проекта

Выполните следующие шаги по порядку:

### 1. Обновление проекта

Подтяните последние изменения из репозитория:

```bash
git pull
```

### 2. Установка зависимостей

Установите нужные пакеты:

```bash
npm install
```

### 3. Подготовка базы данных

Примените миграции (для создания таблиц) и заполните БД начальными данными:

```bash
npx prisma migrate dev
npx prisma db seed
```

### 4. Запуск приложения

Запустите сервер для локальной разработки:

```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:3000

## Структура проекта

Для того чтобы вам было проще ориентироваться, ниже описано, за что отвечает каждая папка в проекте:

- 📂 **`app/`** — **Фронтенд (клиентская часть пользователя, интерфейс)**
    - `app/pages/` — файловая маршрутизация для создания страниц в приложении.
      Файл app/pages/index.vue соответствует маршруту /
      Файл app/pages/login.vue соответствует маршруту /login.
      Файл app/pages/orders/index.vue соответствует маршруту /orders.
    - `app/components/` — Переиспользуемые элементы интерфейса (кнопки, карточки и т.д.).
    - `app/assets/` — Глобальные стили (`main.css`), шрифты.

- 📂 **`server/`** — **Бэкенд (серверная часть API)**
    - `server/api/` — API-роуты. Здесь пишется серверная логика (например, обработка запросов на вход, получение или сохранение данных пользователя из БД).

- 📂 **`prisma/`** — **База данных**
    - `prisma/schema.prisma` — Описание таблиц базы данных. Если нужно добавить новую колонку, связи или таблицу, редактируйте этот файл (после изменений не забудьте выполнить `npx prisma migrate dev`).
    - `prisma/seed.ts` — Файл для скрипта, который заполняет базу `demo_exam` начальными тестовыми данными при запуске `npx prisma db seed`.

## Что такое `.vue` файлы и как с ними работать

Весь интерфейс строится из независимых блоков (компонентов) — файлов с расширением `.vue`.
Каждый такой файл содержит в себе всё необходимое для работы (вёрстку, логику и дизайн) и состоит из трёх основных секций:

1. **`<template>` (Разметка)** — Здесь пишется обычный HTML код (заголовки, кнопки, формы и т.д.). Сюда же можно добавлять классы _TailwindCSS_.
2. **`<script setup>` (Логика)** — Здесь пишется код на JavaScript или TypeScript. В этом блоке мы создаем переменные, пишем функции (например, "что будет, если нажать на кнопку") и обращаемся к серверу за данными.
3. **`<style scoped>` (Внешний вид)** — Здесь пишутся CSS-стили. Благодаря слову `scoped`, стили применятся **только** к этому компоненту и не сломают дизайн на других страницах! Это опциональный блок, и его можно удалить, если вы используете только _TailwindCSS_.

### Где лучше менять стили?

- **Глобальные стили (на всём сайте)** — основные цвета, шрифты и переменные меняются в `app/assets/main.css`.
- **Локальные стили (на одной странице или кнопке)** — меняются:
    - Прямо в тегах через классы Tailwind (например, `<button class="bg-red-500 rounded-md">`);
    - Либо в блоке `<style scoped>` внутри `.vue`-файла этой страницы или компонента.

## Tailwind CSS

В проекте используется встроенная библиотека [Tailwind CSS](https://tailwindcss.com/) для стилизации.

**Что это такое?**
Вместо того чтобы придумывать названия классов и писать свойства в отдельном `.css` файле, мы применяем готовые классы-утилиты (от Tailwind) прямо внутри HTML-тегов (`<template>`).

**Примеры применения:**

- Цвет текста: `text-red-500` (красный текст), `text-white` (белый текст).
- Фон: `bg-green-500` (зелёный фон), `bg-blue-900` (глубокий синий фон).
- Отступы (margin / padding): `mt-4` (margin-top: 1rem), `p-5` (padding: 1.25rem).
- Скругления: `rounded-md` (средние закругления углов), `rounded-full` (круглый элемент).

**Как это выглядит в коде:**

```html
<!-- Обычная кнопка: зеленый фон, белый текст, скругленные углы, внутренние отступы -->
<button class="bg-green-500 text-white px-4 py-2 rounded-md">Сохранить</button>

<!-- Красный текст с отступом сверху -->
<p class="text-red-500 mt-4">Произошла ошибка!</p>
```
